Account Commands

register

The ‘register’ command enables user registration through the command line. It sends a POST request to the ‘/register’ endpoint with the provided username and password.

Usage
$ BSS:::>register -u <username> -p <password>
Command Arguments

-u, --username: Specifies the desired username for registration.
-p, --password: Specifies the desired password for registration.

Example Usage
$ BSS:::>register -u dela_cruz -p strongPass123
User "dela_cruz" registered successfully

login

The ‘login’ command handles user authentication by validating the provided username and password. It send POST request to the ‘/login’ endpoint with the provided username and password to validate.

Usage
$ BSS:::>login -u <username> -p <password>
Command Arguments

-u, --username: User's username
-p, --password: User's password

Example Usage
$ BSS:::>login -u dela_cruz -p strongPass123
Login successful

logout

The ‘logout’ command is designed to handle POST requests for user logout. If the user is currently logged in, it clears the session, logging the user out successfully. It sends a POST request to the API endpoint, handles the response, and updates the CLI session accordingly. Upon successful logout, the session cookie is cleared, and the CLI user details are reset.

Usage
$ BSS:::>logout
Command Arguments

-u, --username: User's username
-p, --password: User's password

Example Usage
$ BSS:USER:dela_cruz:>logout
Login successful

change_username

The ‘change_username’ command allows users to change their username by sending a POST request to the ‘/change_username’ endpoint on the server. The command takes a single argument, <new_username>, representing the desired new username.

Usage
$ BSS:::>change_username -n <new_username>
Command Arguments

-n, --new_username : Specifies the desired new username.

Example Usage
$ BSS:USER:dela_cruz:> -n john_mark
Username changed successfully

change_password

The ‘change_password’ command allows the logged-in user to change their password by sending a POST request to the ‘/change_password’ endpoint. The new password is specified as an argument when using the command.

Usage
$ BSS:::>change_password -p <new_password>
Command Arguments

-p, --new_password: Specifies the desired new password.

Example Usage
$ BSS:USER:john_mark:>change_password -p newPass123
Password changed successfully

delete_account

The delete_account command allows the user to delete their account by sending a DELETE request to the ‘/delete_account’ endpoint. This action permanently removes the user's account.

Usage
$ BSS:::>delete_account
Example Usage
$ BSS:USER:john_mark:>delete_account
User account deleted successfully. You have been logged out

Manage Blockchain

create

The ‘create’ command allows a logged-in user to create a new blockchain by sending a CREATE request to the /create_blockchain endpoint . The blockchain is associated with the user and stored in a dedicated table in the database.

Usage
$ BSS:::>create -bn <blockchain_name> -pub <boolean> -pass <blockchain_password>
Command Arguments

-bn, --blockchain_name : Specifies the desired new blockchain name.
-pub, --new_username : Specifies the desired new username.
-pass, --new_username : Specifies the desired new username.

Example Usage
$ BSS:USER:dela_cruz:>create -bn students -pub 0 -pass strongPass123
Blockchain "students" created successfully

delete_blockchain

The ‘delete_blockchain’ command is used to delete an existing blockchain. The CLI command delete_blockchain interacts with this endpoint by sending a DELETE request with the specified blockchain name.

Usage
$ BSS:::>delete_blockchain -bn <blockchain_name>
Command Arguments

-bn, --blockchain name: Identifies the intended blockchain

Example Usage
$ BSS:USER:dela_cruz:>delete_blockchain -bn age
Blockchain "age" deleted successfully

verify_blockchain

The ‘verify_blockchain’ command allows users, both regular and admin, to verify the integrity of a blockchain. The verification process involves checking the consistency and validity of the blocks within the blockchain.

Usage
$ BSS:::>verify_blockchain -bn <blockchain_name>
Command Arguments

-bn, --blockchain name: Identifies the intended blockchain

Example Usage
$ BSS:USER:dela_cruz:>verify_blockchain -bn sex
Blockchain integrity verified

select_blockchain

The ‘select_blockchain’ command allows users to select a blockchain for further interactions. It sends a request to the server to choose a specific blockchain by name.

Usage
$ BSS:::>select_blockchain -bn <blockchain_name>
Command Arguments

-bn, --blockchain name: Identifies the intended blockchain

Example Usage
$ BSS:USER:dela_cruz:>select_blockchain -bn age
Blockchain "age" has been selected

deselect

The ‘deselect’ command is used to deselect or clear the currently selected blockchain. After calling this method.

Usage
$ BSS:::>deselect
Command Arguments

-bn, --blockchain name: Identifies the intended blockchain

Example Usage
$ BSS:USER:dela_cruz:>deselect
Blockchain "age" has been deselected

Store Data

Store

The store command allows users to store data in a specified blockchain. The user must be logged in, own the specified blockchain, and provide the necessary information, including the blockchain name, data to store, and a reference for the stored data.

Usage
$ BSS:::>store -bn <blockchain_name> -d <data> -r <reference>
Command Arguments

-bn, --blockchain name: Identifies the intended blockchain.
-d, --data: Specifies the information to be stored.
-r, --reference: Specifies the reference to associate with the stored data.

Example Usage
$ BSS:USER:dela_cruz:>store -bn students -d john_doe -r names
Data stored in "students"

Store Hash

The store_hash endpoint allows users to store doubly hashed data in a specified blockchain. The user must be logged in, own the specified blockchain, and provide the necessary information, including the blockchain name, data to be doubly hashed, and a reference for the stored doubly hashed data.

Usage
$ BSS:::>store_hash -bn <blockchain_name> -d <data> -r <reference>
Command Arguments

-bn, --blockchain_name: Name of the blockchain where data will be stored (optional, uses the selected blockchain if not provided)
-data, --data: Data to be doubly hashed and stored in the blockchain
-ref, --reference: Reference for the stored doubly hashed data

Example Usage
$ BSS:USER:dela_cruz:>store_hash -bn students -d john_doe -r names
Hashed data stored in "students"

Search Blocks

Search Blocks

The search_data allows users to search for data in a specified blockchain based on given criteria and a search value. The user must be logged in, own the specified blockchain, and provide the necessary information, including the blockchain name, search criteria, and search value.

Usage
$ BSS:::>search_data -bn <blockchain_name> -c <criteria> -v <value>
Command Arguments

-bn, --blockchain name: Name of the blockchain where data will be searched (optional, uses the selected blockchain if not provided)
-c, --criteria: Criteria for searching (e.g., column name)
-v, --value: Value to search for in the specified criteria

Example Usage
$ BSS:USER:dela_cruz:>search_data -bn students -c reference -v names
3 matching data found in the blockchain. Display 3 out 3
[2] john_doe
[3] mark_davis
[4] ben_trum

display_search_result

The ‘dispaly_search_results’ returns the search results in a JSON format. If the search results are empty, it returns a message indicating that the search results are empty. The CLI command display_search_results interacts with this endpoint, sending a GET request to retrieve and display the search results.

Usage
$ BSS:::>display_search_results
Example Usage
$ BSS:USER:dela_cruz:>display_search_results
[[2] john_doe, [3] mark_davis, [4] ben_trum]

get_search_index

The ‘get_search_index’ command retrieves an element from the search index based on the provided index. The CLI command get_search_index interacts with this endpoint, sending a GET request with the specified index.

Usage
$ BSS:::>get_search_index --index<index>
Command Arguments

--index: The index of the element to retrieve from the search index

Example Usage
$ BSS:USER:dela_cruz:>display_search_results
[[2] john_doe, [3] mark_davis, [4] ben_trum]

Update Blocks

update_by_id

The update_by_id command lets authenticated users modify a blockchain block's data using its unique identifier, with ownership and API key validations included in the process. The response provides success messages or specific error responses based on the request outcome.

Usage
$ BSS:::>update_by_id -bn <blockchain_name> -id <id_number> -n <new_data>
Command Arguments

-bn, --blockchain name:
-id, --id number:
-n, --new data:

Example Usage
$ BSS:USER:dela_cruz:>update_by_id -bn students -id 2 -n anna_gab
Block ID 2 update successfully

update_block

The update_block command allows users to update a block in a specified blockchain based on given criteria and a search value. The user must be logged in, own the specified blockchain, and provide the necessary information, including the blockchain name, update criteria, search value, and the new data.

Usage
$ BSS:::>update_block -bn <blockchain_name> -c <criteria> -v <value> -n <new_data>
Command Arguments

-bn, --blockchain_name: Name of the blockchain where data will be updated (optional, uses the selected blockchain if not provided)
-c, --criteria: Criteria for updating the block (e.g., column name)
-v, --value: Value to search for in the specified criteria to identify the block to be updated
-n, --new_data: New data to replace the existing data in the identified block

Example Usage
$ BSS:USER:dela_cruz:>update_block -bn students -c id -v 1 -n anna_gab
Block ID 2 update successfully

Delete Blocks

delete_block

The delete_block command allows users to delete a blockchain and all associated data by specifying the blockchain name. It performs the blockchain deletion based on the provided blockchain name.

Usage
$ BSS:::>delete_block -bn <blockchain_name>
Command Arguments

-bn, --blockchain_name: Name of the blockchain to be deleted (optional, uses the selected blockchain if not provided)

Example Usage
$ BSS:USER:dela_cruz:>delete_block -bn students
Block ID 2 update successfully

delete_blocks_by_criteria

The delete_blocks_by_criteria allows users to delete references in a specified blockchain based on given criteria and a search value. The user must be logged in, own the specified blockchain, and provide the necessary information, including the blockchain name, delete criteria, and delete value. It sends a DELETE request to the API endpoint, and the response is handled appropriately, printing success messages or error details.

Usage
$ BSS:::>delete_block -bn <blockchain_name> -c <criteria> -v <value>
Command Arguments

-bn, --blockchain_name: Name of the blockchain where references will be deleted (optional, uses the selected blockchain if not provided)
-c, --criteria: Criteria for deleting references (e.g., column name)
-v, --value: Value to search for in the specified criteria to identify the references to be deleted

Example Usage
$ BSS:USER:dela_cruz:>delete_block -bn students
Block ID 2 update successfully

Lists

list

The ‘list’ command allows users to retrieve a list of blockchains associated with their account. The user must be logged in and provide valid authentication, and the command fetches and displays the blockchains owned by the user.

Usage
$ BSS:::>list
Example Usage
$ BSS:USER:dela_cruz:>list
Blockchains:
- students
- accounts

list_r

The ‘list_r’ command enables users to fetch a list of references associated with their account. To utilize this command, users need to be logged in and provide valid authentication. Upon execution, the command retrieves and displays the blockchains owned by the user.

Usage
$ BSS:::>list_r
Example Usage
$ BSS:USER:dela_cruz:>list_r
References:
- password
- names
- Genesis

API Commands

list

The ‘apikey_gen’ command allows logged-in users to generate a new API key. The generated key is associated with the user's account and can be optionally given a name (API name). The API key is then stored in the database for later use.

Usage
$ BSS:::>apikey_gen -n <api_name>
Command Arguments

-n or --api_name (optional): Specifies the name to be associated with the generated API key. If not provided, the default name is "Secret Key."

Example Usage
$ BSS:USER:dela_cruz:>apikey_gen -n dela_cruz
Generated API Key for dela_cruz: 219c1eff-89a5-4f4d-9e4f-626ea2249fd0

set_key

The ‘set_key’ command allows users to change the API key used for making requests to the blockchain server. The command takes a new API key as an argument and updates the API key associated with the CLI session.

Usage
$ BSS:::>set_key -n <new_key>
Command Arguments

-n, --new_key: New API key to be set

Example Usage
$ BSS:USER:dela_cruz:>set_key -n 219c1eff-89a5-4f4d-9e4f-626ea2249fd0
Key updated successfully.

show_key

The ’show_key’ command enables users to view the current API key associated with the CLI session. When the command is executed, it prints the API key to the console.

Usage
$ BSS:::>show_key
Command Arguments

-n, --new_key: New API key to be set

Example Usage
$ BSS:USER:dela_cruz:>show_key
Key: 219c1eff-89a5-4f4d-9e4f-626ea2249fd0

revoke_key

The ‘revoke_key’ command allows users to revoke an API key associated with their account for a specified API name. The CLI command revoke_key interacts with this endpoint, sending a POST request with the API name to be revoked.

Usage
$ BSS:::>show_key -n <new_key>
Command Arguments

-n, --new_key: New API key to be set

Example Usage
$ BSS:USER:dela_cruz:>show_key -n dela_cruz
API Key for dela_cruz revoked successfully
